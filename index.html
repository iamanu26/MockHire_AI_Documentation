01 — OBJECTIVE

Why MockHire AI Exists

Most platforms give you text-based Q&A. We give you an actual interviewer.

✗

No Realistic Practice

Text-based tools can't simulate the pressure of speaking answers aloud in real time under interview conditions.

⊘

No Measurable Feedback

Generic tips don't tell you where you actually failed. Candidates need scores on real dimensions from real answers.

⊞

Disconnected DSA Prep

Coding practice and interview simulation live on separate platforms. MockHire unifies them into one workflow.

⟳

Static Question Banks

Memorizable lists don't prepare you for real interviewers. You need AI-generated, conversation-aware follow-ups.

◈ MISSION STATEMENT

To create a comprehensive, AI-driven platform that simulates real interview conditions — enabling candidates to practice speaking answers aloud, receive intelligent follow-up questions, and get scored, actionable feedback. All in one place.

02 — FEATURES

Four Core Modules

Each module designed to solve a specific gap in how candidates prepare.

01

Voice Interview Simulation

AI asks questions aloud via speech synthesis. You answer verbally. The browser's Web Speech API captures every word and sends it to LLaMA 3.1 for intelligent follow-up generation.

  • Conversation-aware follow-ups
  • Session isolation per interview
  • Real-time voice capture

After ending a session, get a detailed performance scorecard generated directly from your conversation history.

LeetCode-style coding environment with AI-generated problems and expert review of your solutions.

User Auth & History

JWT-based authentication keeps your interview records private and persistent. Track improvement over time.

  • JWT login & registration
  • Per-user session history
  • Persistent score records
  • Secure endpoint protection

06 — ENGINEERING

Key Challenges Solved

Real bugs. Real fixes. Engineering decisions that made the platform production-ready.

BUG FIX

Shared AI Agent Instance

⚠ Problem

Two separate InterviewAgent objects were created — one for conversation, one for feedback — causing the feedback report to generate on an empty history, returning meaningless scores.

✓ Solution

Refactored to a single shared agent instance per session. Feedback now always reflects the full conversation history, guaranteeing accurate scoring.

PARSER

JSON Parsing Robustness

⚠ Problem

LLaMA sometimes returns scores as "7/10" strings, or wraps JSON in markdown fences like ```json ... ```, breaking all standard parsers.

✓ Solution

Built a custom multi-format parser that handles raw JSON, markdown-wrapped JSON, and fractional score strings uniformly — with a server-side type coercion safety net.

SESSION

Session Isolation

⚠ Problem

Without session clearing between runs, follow-up questions and scores would bleed across multiple interview sessions for the same user — contaminating history.

✓ Solution

Added a dedicated /interview/start endpoint that clears conversation history before each session. Every session is now independently scored.

PROMPT ENG.

Strict AI Evaluation

⚠ Problem

Without scoring constraints, LLaMA would inflate scores for minimal effort answers — making feedback feel rewarding but completely useless for real preparation.

✓ Solution

Engineered prompts with explicit scoring rubrics and hard ceiling rules. Added a server-side score cap as a final safety net against overly generous LLM responses.
