01 — OBJECTIVE

Why MockHire AI Exists

Most platforms give you text-based Q&A. We give you an actual interviewer.

✗

No Realistic Practice

Text-based tools can't simulate the pressure of speaking answers aloud in real time under interview conditions.

⊘

No Measurable Feedback

Generic tips don't tell you where you actually failed. Candidates need scores on real dimensions from real answers.

⊞

Disconnected DSA Prep

Coding practice and interview simulation live on separate platforms. MockHire unifies them into one workflow.

⟳

Static Question Banks

Memorizable lists don't prepare you for real interviewers. You need AI-generated, conversation-aware follow-ups.

◈ MISSION STATEMENT

To create a comprehensive, AI-driven platform that simulates real interview conditions — enabling candidates to practice speaking answers aloud, upload their resume for personalized questions, receive intelligent follow-up questions, get scored actionable feedback, and consult an AI interview coach. All in one place, secured by OAuth, JWT, and email verification.

06 — ENGINEERING

Key Challenges Solved

Real bugs. Real fixes. Engineering decisions that made the platform production-ready.

BUG FIX

Shared AI Agent Instance

⚠ Problem

Two separate InterviewAgent objects were created — one for conversation, one for feedback — causing the feedback report to generate on an empty history.

✓ Solution

Refactored to a single shared agent instance per session. Feedback now always reflects the full conversation history, guaranteeing accurate scoring.

PARSER

JSON Parsing Robustness

⚠ Problem

LLaMA sometimes returns scores as "7/10" strings, or wraps JSON in markdown fences like ```json ... ```, breaking all standard parsers.

✓ Solution

Built a custom multi-format parser that handles raw JSON, markdown-wrapped JSON, and fractional score strings uniformly — with a server-side type coercion safety net.

NEW

PostgreSQL Migration

⚠ Problem

SQLite's single-writer lock blocked concurrent sessions and lacked the connection pooling needed for multi-user production loads.

✓ Solution

Migrated to PostgreSQL via asyncpg driver with Alembic migrations. Added a RESUMES table and updated all foreign keys. Zero data loss via migration scripts.

NEW

OAuth + Email Verification

⚠ Problem

Supporting both password-based and Google OAuth sign-in with a unified session model required handling two completely different identity flows.

✓ Solution

Added oauth_provider and is_verified columns to the USERS table. Email OTP on registration, Google callback on OAuth. Both paths issue the same JWT payload.

NEW

Resume Context Injection

⚠ Problem

Generic AI questions don't reflect a candidate's actual background. Uploading a PDF and injecting its content into every prompt introduced token-limit and latency concerns.

✓ Solution

PyMuPDF extracts resume text, LLaMA summarizes it to a compact skills JSON, stored in PostgreSQL. Only the JSON summary is injected into the interview system prompt — minimal tokens, maximum relevance.

PROMPT ENG.

Strict AI Evaluation

⚠ Problem

Without scoring constraints, LLaMA would inflate scores for minimal effort answers — making feedback feel rewarding but completely useless for real preparation.

✓ Solution

Engineered prompts with explicit scoring rubrics and hard ceiling rules. Added a server-side score cap as a final safety net against overly generous LLM responses.

How do I answer "tell me about yourself" for a backend engineer role?
Coach

Great question. For a backend role, structure it using the Present → Past → Future framework:

Present: State your current role & main tech stack (FastAPI, Postgres, etc.)

Past: Mention 1-2 key projects with impact metrics

Future: Tie your goal to the role you're applying for

💡 Keep it under 90 seconds. Practice aloud using MockHire's voice mode.

Coach

MockHire scores Communication on clarity, structure, and conciseness. Aim for 7+ / 10 as a target. Below 6 usually means answers are too vague or too long.
